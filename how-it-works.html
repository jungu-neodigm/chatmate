01

프로젝트 생성

먼저 브랜드 또는 조직 단위의 챗봇 프로젝트를 생성합니다. 홈페이지 주소와 허용 도메인을 등록해 승인된 웹사이트에서만 챗봇이 동작하도록 준비합니다.

입력 정보

  • 프로젝트명 / 고객사명
  • 대표 홈페이지 주소
  • 허용 도메인
  • 담당자 이메일
  • 사용 모델 / 월 사용 한도

프로젝트 상태값

준비중 설정중 검수중 운영중 일시중지 종료
02

관리자 및 권한 설정

관리자와 운영 담당자를 초대해 역할별로 접근 권한을 나눌 수 있습니다. 지식 데이터 관리, 답변지침 수정, 질문 리스트 확인 등 업무에 맞는 권한을 부여합니다.

03

지식 데이터 연결

웹사이트, FAQ, 문서, 상품 정보 등 고객이 자주 찾는 정보를 챗봇의 지식 데이터로 연결합니다. AI는 이 데이터를 기준으로 답변하므로, 기존 콘텐츠를 대화형 고객 경험으로 전환할 수 있습니다.

데이터 유형

웹사이트 URL · 사이트맵 · FAQ · 직접 Q&A · 상품 정보 · 서비스 소개 문서 · 도움말 문서 · 업무 매뉴얼 · 정책 문서 · 공지사항

관리 항목

데이터명 · 유형 · 출처 URL · 카테고리 · 우선순위 · 사용 상태 · 처리 상태 · 마지막 처리일 · 실패 사유 · 재처리

04

데이터 처리 및 검수

수집된 데이터는 자동으로 정제되고 검색 가능한 형태로 처리됩니다. 처리 실패나 제외 대상은 사유를 확인할 수 있어, 운영자가 어떤 데이터를 보강해야 하는지 쉽게 파악할 수 있습니다.

등록→ 수집→ 본문 추출/정제→ 중복 제거/분할→ 임베딩→ 완료

실패 사유 그룹

URL 분류

학습 가능 · 검색 전용 · 제외 · 검토 필요

05

챗봇 화면 설정

브랜드나 서비스 톤에 맞게 챗봇의 첫인상과 대화 화면을 설정합니다. 첫 메시지, 추천질문, 입력창 문구, 아이콘, 하단 고지문까지 관리자가 직접 조정할 수 있습니다.

설정 항목

플로팅 버튼 아이콘 · 채팅창 로고/이름 · 홈페이지 주소 · AI봇 아이콘 · 첫 메시지 · 입력창 안내문 · 하단 고지문 · 추천질문 · 글자 크기

사용자 화면 기능

플로팅 버튼 · 첫 메시지·추천질문 노출 · 텍스트 질문 입력 · 출처 표시 · 좋아요/싫어요 평가 · 대화 초기화 · 글자 크기 확대

06

답변 정책 설정

AI가 브랜드에 맞지 않는 답변을 하지 않도록 답변지침과 안전 정책을 설정합니다. 기업별 말투, 금지 표현, 개인정보 처리 기준, 근거 부족 시 응답 방식을 관리할 수 있습니다.

기본 답변 정책

등록된 지식 데이터 우선 검색 · RAG 기반 답변 · 근거 부족 시 임의 답변 금지 · 출처 제공 · 개인정보 포함 질문 차단 · 모델 정보 미노출 · 브랜드 톤 유지

07

테스트 검수 및 배포

운영 전 테스트 질문으로 답변 품질과 사용자 화면을 검수합니다. 검수가 완료되면 제공된 위젯 스크립트를 웹사이트에 삽입해 챗봇을 공개합니다.

검수 항목

첫 메시지/추천질문 노출 · 정상 질문 답변 · 출처 표시 · RAG 실패 응답 · 개인정보 차단 · 부정 평가 저장 · 모바일 반응형 · 도메인 검증

위젯 배포

위젯 스크립트 발급 → 웹사이트에 script 삽입 → siteId 기반 초기화 → 허용 도메인 검증 → 운영중 전환 → 실제 노출

08

운영 분석 및 개선

챗봇이 실제로 받은 질문과 답변 품질 데이터를 분석해 지속적으로 개선합니다. 반복 질문은 추천질문으로 등록하고, 실패 질문은 직접 Q&A나 지식 데이터 보강으로 해결할 수 있습니다.

운영 지표

질문 수 · 답변 성공률 · RAG 실패 수 · 부정 평가 수 · 개인정보 차단 수 · CTA 클릭 수 · 토큰 사용량 · 예상 비용

개선 방식

직접 Q&A 등록 · 기존 답변 수정 · 출처 URL 추가 · 추천질문 등록 · 답변지침 수정 · 지식 데이터 재처리
